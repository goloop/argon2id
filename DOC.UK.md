## Огляд

Argon2id - це гібридний варіант, рекомендований для зберігання паролів: він
поєднує data-independent адресацію Argon2i (стійку до таймінг-каналів) для
першої половини першого проходу з data-dependent адресацією Argon2d (стійкою до
time-memory trade-off) для решти. Він memory-hard, тож зловмисник, який викрав
хеш, змушений витрачати реальну RAM на кожну спробу підбору офлайн.

Пакет реалізує Argon2id і його внутрішній BLAKE2b (RFC 7693) напряму на
стандартній бібліотеці, без стороннього коду.

## Хешування й перевірка

```go
h := argon2id.New()

encoded, err := h.Hash([]byte(password)) // зберігаєте encoded
err = h.Verify(encoded, []byte(attempt)) // nil при збігу
```

`Hash` бере свіжу випадкову сіль із системного CSPRNG (`crypto/rand`) і повертає
самоописовий рядок. `Verify` зчитує з цього рядка параметри й сіль, переобчислює
дайджест і порівнює за сталий час (`crypto/subtle`). Невірний пароль повертає
`ErrMismatch`; спотворений чи позазмежний хеш - `ErrInvalidHash`.

## Параметри вартості

```go
h := argon2id.New(
	argon2id.WithMemory(128*1024), // KiB (128 МіБ)
	argon2id.WithTime(3),          // проходи по памʼяті
	argon2id.WithThreads(4),       // смуги (паралелізм)
	argon2id.WithSaltLength(16),   // байтів випадкової солі
	argon2id.WithKeyLength(32),    // байтів дайджесту
)
```

| Опція | Дефолт | Значення |
|---|---|---|
| `WithMemory(kib)` | 65536 (64 МіБ) | вартість памʼяті в KiB |
| `WithTime(passes)` | 2 | кількість проходів по памʼяті |
| `WithThreads(lanes)` | 1 | паралелізм (смуги) |
| `WithSaltLength(n)` | 16 | довжина випадкової солі в байтах |
| `WithKeyLength(n)` | 32 | довжина дайджесту в байтах |

Більше памʼяті й часу роблять кожен хеш повільнішим, а кожну спробу підбору -
дорожчою; правильні числа залежать від вашого бюджету латентності й заліза, тож
вимірюйте. Кожне значення затиснуте в безпечний діапазон.

## Кодування

`Hash` повертає стандартний PHC-рядок:

```text
$argon2id$v=19$m=65536,t=2,p=1$<base64-сіль>$<base64-дайджест>
```

Сіль і дайджест - `base64.RawStdEncoding` (без паддінгу). Зберігайте весь рядок в
одній колонці; окремих параметрів вести не треба.

## Перехешування

`NeedsRehash` повідомляє, чи збережений хеш слабший за поточні налаштування
хешера, щоб успішний вхід міг прозоро його підсилити:

```go
if err := h.Verify(stored, pw); err == nil {
	if h.NeedsRehash(stored) {
		fresh, _ := h.Hash(pw) // зберігаєте fresh замість stored
	}
}
```

Повертає true для спотвореного хеша або такого, чиї памʼять, час, сіль чи довжина
ключа нижчі за поточні налаштування.

## Межі й валідація

Кодований хеш повністю валідується до будь-якої алокації памʼяті. Парсер
відкидає невірний алгоритм чи версію, поле параметрів із зайвим «сміттям», і
будь-яку вартість поза цими межами:

| Параметр | Мін | Макс |
|---|---|---|
| памʼять | 8 x смуг KiB | 1 ГіБ |
| час | 1 | 64 |
| смуги | 1 | 255 |
| сіль | 8 байтів | 64 байти |
| дайджест | 16 байтів | 128 байтів |

Довжини полів перевіряються до base64-декодування, тож завелике поле не змусить
велику алокацію. Стелі операційні: вони сидять значно вище за будь-яку розумну
вартість, водночас обмежуючи те, що один `Verify` може змусити витратити.

## Коректність

Реалізація закріплена за опублікованими тест-векторами, які проганяє `go test`:

- Argon2id - вектор RFC 9106 (пароль, сіль, секрет і асоційовані дані).
- BLAKE2b - вектори RFC 7693, зокрема порожній ввід.

Вектор, що проходить, означає, що конструкція збігається з референсом біт-у-біт.

## Залежності

Немає. Лише стандартна бібліотека.

## Межі застосування

Цей пакет хешує й перевіряє паролі. Він не веде користувачів, сесії чи токени;
для токенів, ротації refresh і HTTP-middleware використовуйте
[`goloop/auth`](https://github.com/goloop/auth), чий інтерфейс `PasswordHasher`
цей пакет задовольняє.
